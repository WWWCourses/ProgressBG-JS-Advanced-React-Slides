Why JSX?

Why JSX?

Demo

You can create elements either using pure JavaScript syntax or by using the JSX syntax
JSX just provides syntactic sugar for the React.createElement(component, props, ...children) function.
In next example both React elements are rendered to identical DOM structure:

  		const jsElement = React.createElement("div", {className: "App"},
  		  React.createElement("h1", null,
  		    React.createElement('span',null,'Hello')));
  	

  		const jsxElement =
  		  <div className="App">
  		      <h1><span>Hello</span></h1>
  		  </div>
the jsxElement is transpiled by babel to the same code, as in jsElement: babel repl

JSX in HTML?

JSX in HTML?

If you want to use JSX syntax in script element inside a HTML file, you have to import the babel (as JSX is compiled to JavaScript by babel) and set the script attribute type="text/babel"
Of course, this is not recommended for production!

			<!DOCTYPE html>
			<html lang="en">
			<head>
				<!-- use latest react version -->
				<script type="text/javascript" src="https://unpkg.com/react@latest/umd/react.production.min.js"></script>
				<script type="text/javascript" src="https://unpkg.com/react-dom@latest/umd/react-dom.production.min.js"></script>

				<!-- use latest babel -->
				<script src="https://unpkg.com/@babel/standalone/babel.min.js"></script>
			</head>
			<body>
				<div id="root"></div>

				<script type="text/babel">
					let reactJSXElement = <h1 className="green">Hello, world!</h1>

					let rootDOMElement = document.querySelector('#root');

					ReactDOM.render(reactJSXElement, rootDOMElement);
				</script>
			</body>
			</html>

Preventing Injection Attacks

Preventing Injection Attacks

By default, React DOM escapes any values embedded in JSX before rendering them
Everything is converted to a pure string before being rendered
This helps prevent Injection and XSS (cross-site-scripting) attacks.
Of course, always double check, because Avoiding XSS in React is Still Hard
