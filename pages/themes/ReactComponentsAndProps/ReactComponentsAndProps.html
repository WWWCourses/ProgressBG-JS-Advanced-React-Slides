Components

Components

Overview

Components are the building blocks of any React app.
Thinking in components allows us to split the UI into independent, reusable pieces, and work with each piece in isolation.
A React Component is a JavaScript class or function that optionally accepts inputs i.e. properties(props) and returns a React Element describing what should appear on the screen.
React lets you define components as classes or functions.

Class Component

Class Component

Overview

From ES6, we can use class syntax to define a component:
Before that, you could've use the React's createClass method
The only method you must define in a React.Component subclass is the render() method

      //class component, without using "props":
      class HelloWorld extends React.Component {
        render() {
          return 
Hello World!
;
        }
      }

Accessing props in Class Component

the "props" object is available to any Class Component as property to the class. I.e. with this.props

{this.props.title}

Component inheritance?

All the internal React Component logic comes from the extends React.Component.
However, it isn’t recommended to use the concept of inheritance for your custom class components more than that
In React code reuse is primarily achieved through composition rather than inheritance

Rendering a Component

Rendering a Component

React elements can represent DOM tags or user-defined components

      // element, representing a DOM tag:
      const element = <div />;

      // element, representing a user-defined component:
      const element = <HelloWorld />;

Naming conventions

User-Defined Components Must Be Capitalized
React treats components starting with lowercase letters (like '<div />') as DOM tags.
And components starting with capital letter as user-defined, which have to be declared in the scope
Reference: React treats components starting with lowercase letters as DOM tags

Props

Props

Overview

When React sees an element representing a user-defined component, it passes JSX attributes to this component as a single object. We call this object “props”.

      // Welcome component returns a React element as the result:
      const Welcome = (props) => <h1>Hello {props.name}!</h1>;

      // React calls the Welcome component with {name: 'Maria'} as the props.
      const element = <Welcome name="Maria" />;

      // We call React DOM, which updates the DOM to match <h1>Hello, Maria</h1>
      ReactDOM.render(
        element,
        document.getElementById('root')
      );

Overview

Props are way of passing data from parent to child component
React follows the rule that properties have to flow down from a parent component to an immediate child component.
React implements only one-way data binding!

Props propagation

If we want to send props from component 1 to component 6, every child component in this path has to receive the property from its parent and then resend that property to its child

Props immutability

Props are immutable objects, since they are passed from parent to child!
