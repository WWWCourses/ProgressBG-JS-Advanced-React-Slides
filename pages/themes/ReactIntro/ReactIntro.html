What is React

React is a view (UI) library, not (yet another) SPA framework
Yet the whole ecosystem around React makes it possible, and incrementally easy, to build SPA applications.
It only enables you to render components as viewable elements in a browser
It is the V in the MVC (Model View Controller pattern).
But the view is a hierarchy of composable components.

The simplest "hello world" react app, using CDN

React is just a library, so to get started simple, it's enough only to include it and use it

      <head>
        <title>Simplest React App</title>
        <script type="text/javascript" src="https://unpkg.com/react@latest/umd/react.production.min.js"></script>
        <script type="text/javascript" src="https://unpkg.com/react-dom@latest/umd/react-dom.production.min.js"></script>
      </head>
      <body>
        <div id="root"></div>

        <script>
          let reactElement = React.createElement("h1", null, "Hello, world!");
          let rootDOMElement = document.querySelector('#root');

          ReactDOM.render(reactElement, rootDOMElement);
        </script>
      </body>
Of course, in practise we use React libraries with NPM!

Init the project

In order to use react libraries as NPM modules we will need a module bundler like webpack
Webpack is a module bundler, it takes a bunch of modules and turns them into a single bundle

      # create the project folder
      mkdir simplest-react-app

      # go into project folder
      cd simplest-react-app/

      # initialise a new node project
      npm init -y

      # install react
      npm install --save-dev react react-dom

      # install webpack
      npm install --save-dev webpack webpack-cli

Project folder structure

Make sure you have next structure:

      ├── dist
      ├── index.html
      ├── package.json
      ├── src
      │   └── index.js
      └── webpack.config.js

Webpack config


      const path = require('path')
      // const HtmlWebpackPlugin = require('html-webpack-plugin')

      module.exports = {
        entry: path.join(process.cwd(), 'src/index'),
        output: {
          path: path.join(__dirname, '/dist'),
          filename: 'index.bundle.js'
        },

        // Rules of how webpack will take our files, complie & bundle them for the browser
        module: {
          rules: [
            {
              test: /\.(js|jsx)$/,
              exclude: /nodeModules/,
            }
          ]
        }
      }

Build the project

We can build our project by starting webpack manually:

      npx webpack --mode production
But it is more convenient to setup this command in pacakge.json

      "scripts": {
        "build": "webpack --mode production"
      }
And then just type:

      npm run build

React Project folder structure

React Project folder structure

React gives you the freedom to organize your project according to your needs (check File Structure @react docs)
For now we'll keep it simple, by putting all of our components into the src/components folder
Each component will reside in: src/components/ComponentName.js

      src/
      ├── App.css
      ├── App.js
      ├── components
      │   ├── Header.js

      │   └── TodoList.js
      ├── index.css
      ├── index.js
      ├── logo.svg
